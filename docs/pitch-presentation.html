Data Model — 5 Core Tables

TablePurposeKey columns
organizationsTenant root; all data scoped hereid, name, plan
projectsOne bedsheet workspace per projectid, org_id, name, pages
elementsCables, closures, splitters, equipmentid, project_id, type, label, position
portsIndividual fiber strands; color + stateid, element_id, index, color_code, occupied
splicesTyped connections between portsid, port_a_id, port_b_id, splice_type, notes

BFS trace walks the splices graph from any port to the end of the path. Every hop is deterministic and queryable — not a visual heuristic.
